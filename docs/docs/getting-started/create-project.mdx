# Building a P2P Pong Game with Pear

This guide walks you through building a fully peer-to-peer Pong game from scratch using the [Pear runtime](https://docs.pears.com). Two players connect directly — no servers, no middlemen. The finished app includes real-time gameplay, a ping indicator, and a local leaderboard that syncs between players.

---

## What You'll Be Building

- **Two-player Pong** over a direct P2P connection
- A **lobby** where one player hosts and shares a room code
- A **leaderboard** backed by Hypercore (an append-only log that syncs over the network)
- A **game loop** that runs on the host and streams state to the guest

### Architecture overview

```
┌──────────────────────────────────────────────────────┐
│  Pear App                                            │
│                                                      │
│  ┌────────────┐   pipe (JSON/newline)  ┌──────────┐ │
│  │  Backend   │ ◄──────────────────── │  UI      │ │
│  │ (index.js) │ ──────────────────── ► │ (app.js) │ │
│  └─────┬──────┘                        └──────────┘ │
│        │ Hyperswarm                                  │
│        │ (direct P2P)                               │
└────────┼─────────────────────────────────────────────┘
         │
    ┌────▼────┐
    │  Peer   │  (the other player's machine)
    └─────────┘
```

The **backend** (`index.js`) handles all networking (Hyperswarm for matchmaking, Hypercore for the leaderboard). The **UI** runs in a browser-like context and talks to the backend through a bidirectional pipe, sending and receiving newline-delimited JSON messages.

## Prerequisites

Install the Pear runtime before starting:

```bash
npm install -g pear
```

## Step 1 — Create the project directory

```bash
mkdir p2p-pong
cd p2p-pong
```

---

## Step 2 — Create `package.json`

```json
{
  "name": "p2p-pong",
  "main": "index.js",
  "type": "module",
  "pear": {
    "pre": "pear-electron/pre",
    "name": "p2p-pong",
    "gui": {
      "main": "ui/index.html",
      "width": 900,
      "height": 600,
      "minWidth": 500
    }
  },
  "license": "MIT",
  "scripts": {
    "dev": "pear run -d ."
  },
  "devDependencies": {
    "pear-interface": "^1.0.0"
  },
  "dependencies": {
    "pear-bridge": "^1.2.0",
    "pear-electron": "^1.7.10",
    "hyperswarm": "^4.7.0",
    "hypercore": "^10.28.0"
  }
}
```

**Key fields explained:**

- `"type": "module"` — enables ES module syntax (`import`/`export`) throughout the project.
- `"pear"` — Pear-specific config. `"pre"` runs `pear-electron` setup before launch. `"gui"` tells Pear to open a window and load `ui/index.html` inside it.
- `hyperswarm` — discovers and connects peers over a distributed hash table (DHT), giving you a direct encrypted connection without a central server.
- `hypercore` — an append-only log that can replicate between peers, used here for the leaderboard.
- `pear-bridge` — provides the message pipe between the backend Node process and the UI.
